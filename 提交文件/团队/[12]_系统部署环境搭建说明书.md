## 二、搭建部署环境和安装软件流程

本流程以 **Windows 10/11** 操作系统为例，从系统初始状态开始，一直到系统能正常运行为止。Linux/macOS 流程类似。

---

### 2.1 安装 JDK 21

**步骤：**

1. 打开浏览器，访问 https://www.oracle.com/java/technologies/downloads/#jdk21-windows
2. 下载 **Windows x64 Installer (.msi)**
3. 双击运行安装程序，全部保持默认选项，点击"下一步"直到完成
4. 配置环境变量（通常安装程序会自动配置）：
   - 如未自动配置，手动添加 `JAVA_HOME` 环境变量，指向 JDK 安装目录（如 `C:\Program Files\Java\jdk-21`）
   - 将 `%JAVA_HOME%\bin` 添加到 `Path` 环境变量
5. **验证安装**：打开命令提示符或 PowerShell，输入：
   ```
   java -version
   ```
   预期输出：
   ```
   java version "21.0.x" 2024-xx-xx LTS
   Java(TM) SE Runtime Environment (build 21.0.x+xx-LTS-xxx)
   Java HotSpot(TM) 64-Bit Server VM (build 21.0.x+xx-LTS-xxx, mixed mode, sharing)
   ```

---

### 2.2 安装 Node.js

**步骤：**

1. 打开浏览器，访问 https://nodejs.org
2. 下载 **LTS 版本**（Windows .msi 安装包）
3. 双击运行安装程序，全部保持默认选项，点击"下一步"直到完成
4. **验证安装**：打开命令提示符或 PowerShell，输入：
   ```
   node -v
   npm -v
   ```
   预期输出（版本号可能略有不同）：
   ```
   v20.x.x
   10.x.x
   ```

---

### 2.3 安装 MySQL 8.0

**步骤：**

1. 打开浏览器，访问 https://dev.mysql.com/downloads/windows/installer/8.0.html
2. 下载 **mysql-installer-community-8.0.x.msi**（约 400MB）
3. 双击运行安装程序：
   - 安装类型选择：**"Server only"**（仅安装数据库服务）
   - 配置类型选择：**"Development Computer"**（开发计算机，占用内存较少）
   - 认证方式：选择 **"Use Strong Password Encryption for Authentication"**
   - 设置 root 密码：**Lazydog@234711**（或自定义密码，需牢记）
   - Windows 服务：保持默认 "MySQL80"，勾选 "Start the MySQL Server at System Startup"
   - 全部点击"下一步"/"Execute"完成安装
4. **验证安装**：
   - 打开"服务"（services.msc），确认 **MySQL80** 服务状态为"正在运行"
   - 或打开命令提示符输入：
     ```
     mysql -u root -p
     ```
     输入密码后能进入 MySQL 命令行即表示成功

---

### 2.4 获取项目代码

**方式一：从 GitHub 克隆（推荐）**

打开命令提示符或 Git Bash，进入要存放项目的目录：

```bash
git clone https://github.com/happystar218/CanteenSimulationsys.git
cd CanteenSimulationsys
```

**方式二：解压部署包**

将 `软件综合实训_12_系统部署包.zip` 解压到目标目录，然后进入解压后的目录：

```bash
cd CanteenSimulationsys
```

---

### 2.5 创建数据库

打开 MySQL 命令行（或使用 MySQL Workbench / Navicat 等工具），执行以下 SQL 语句：

```sql
CREATE DATABASE IF NOT EXISTS canteen_db
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_unicode_ci;
```

**验证数据库创建：**

```sql
SHOW DATABASES;
```

应能看到 `canteen_db` 出现在列表中。

---

### 2.6 配置数据库密码

编辑文件 `canteensimulation/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/canteen_db?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: ${DB_PASSWORD:改成你的MySQL密码}   # ← 修改此处
```

将 `改成你的MySQL密码` 替换为第 2.3 步设置的 root 密码（如 `Lazydog@234711`）。

**注意**：如果使用环境变量，可以设置 `DB_PASSWORD` 环境变量替代明文密码：

```bash
# Windows PowerShell
$env:DB_PASSWORD="你的密码"

# Windows CMD
set DB_PASSWORD=你的密码

# Linux / macOS
export DB_PASSWORD="你的密码"
```

---

### 2.7 启动后端服务

打开命令提示符，进入 `canteensimulation` 目录：

```bash
cd canteensimulation

# Windows
mvnw.cmd spring-boot:run

# Mac / Linux
./mvnw spring-boot:run
```

**首次启动会自动完成以下操作**：
- 创建数据表：`t_seat`（座位表）、`t_window`（窗口表）、`t_simulation_record`（仿真记录表）
- 初始化 514 条座位数据（一楼 280 座 + 二楼 234 座）
- 初始化 16 条窗口数据（按北交大四食堂实际窗口信息）

等待控制台输出以下信息表示启动成功：

```
Started CanteenSimulationApplication in X.XXX seconds (process running for X.XXX)
```

后端默认运行在 **http://localhost:8080**。

---

### 2.8 启动前端服务

**新开一个**命令提示符窗口，进入 `canteen-frontend` 目录：

```bash
cd canteen-frontend

# 首次运行需安装依赖（约 1-2 分钟）
npm install

# 启动开发服务器
npm run dev
```

等待控制台输出：

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

前端默认运行在 **http://localhost:5173/**。

---

### 2.9 验证系统运行

1. 打开浏览器，访问 **http://localhost:5173**
2. 应看到"北交大四食堂仿真监控系统"首页（仿真设置页面）
3. 如果前端能正常显示页面且能加载窗口列表（来自后端 API），说明整个系统部署成功

## 四、常见问题排查

### 问题1：后端启动报 "Access denied for user 'root'@'localhost'"

**原因**：数据库密码配置不正确

**解决**：检查 `application.yml` 中的 `password` 配置是否与安装 MySQL 时设置的密码一致。

### 问题2：后端启动报 "Unknown database 'canteen_db'"

**原因**：未创建数据库

**解决**：执行 `CREATE DATABASE IF NOT EXISTS canteen_db DEFAULT CHARACTER SET utf8mb4;`

### 问题3：前端 `npm install` 报网络错误

**原因**：npm 默认源可能连接不稳定

**解决**：切换为国内镜像源：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 问题4：浏览器访问 localhost:5173 页面空白

**原因**：后端未启动或端口被占用

**解决**：
- 确认后端在 http://localhost:8080 正常运行
- 检查浏览器控制台（F12）是否有网络请求错误
- 确认 `vite.config.js` 中代理配置指向正确的后端地址

### 问题5：端口被占用（8080 或 5173）

**解决**：
- Windows：`netstat -ano | findstr :8080` 找到占用进程 PID，任务管理器结束进程
- Linux/Mac：`lsof -i :8080` 找到进程，`kill -9 <PID>` 结束
