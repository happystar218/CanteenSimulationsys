## 一、如何启动进入系统

### 1.1 前置条件

确保以下服务已正常启动（参考《[12]_系统部署环境搭建说明书》）：

| 服务 | 地址 | 验证方式 |
|------|------|---------|
| MySQL 数据库 | localhost:3306 | 服务 "MySQL80" 正在运行 |
| 后端（Spring Boot） | http://localhost:8080 | 访问 http://localhost:8080/window/list 返回 JSON |
| 前端（Vue 3） | http://localhost:5173 | 浏览器显示系统首页 |

### 1.2 访问系统

1. 打开浏览器（推荐 Chrome、Edge 或 Firefox 最新版本）
2. 在地址栏输入：**http://localhost:5173**
3. 看到"北交大四食堂仿真监控系统"首页，即表示成功进入系统

### 1.3 系统页面导航

系统共有三个主要页面，通过顶部导航栏切换：

| 导航标签 | 路径 | 路由 | 功能 |
|---------|------|------|------|
| 🏠 仿真设置 | `/` | 首页 | 配置仿真参数、管理窗口、控制仿真运行 |
| 📊 仿真监控 | `/monitor` | 监控页 | 实时可视化监控仿真过程 |
| 📋 历史记录 | `/records` | 记录页 | 查看、详情、删除历史仿真记录 |

## 二、各功能界面及使用说明

---

### 2.1 仿真设置页（首页）

**访问路径**：`/`（默认首页）

本页面是系统的主控面板，用于在启动仿真前配置所有参数，并提供窗口管理功能。

#### 2.1.1 仿真参数配置区（页面左半部分）

| 参数 | 可选值 | 默认值 | 说明 |
|------|--------|:----:|------|
| **仿真时段** | 早餐 7:00-9:00 / 午餐 11:00-13:00 / 晚餐 17:00-19:00 | 午餐 | 决定仿真时长 2 小时（7200 仿真秒）。选择不同时段会影响食堂人流模式 |
| **人员流量** | 1 — 240 人/分钟 | 60 | 控制每分钟到达食堂的学生人数。早餐建议 30-80，午餐建议 80-180，晚餐建议 40-100 |
| **速度倍率** | 1x / 5x / 10x / 30x / 60x / 120x | 10x | 仿真时间流逝速度。**1x**=真实速度（2小时完成），**120x**≈1分钟完成。调试时建议低倍率，批量测试时建议高倍率 |
| **到达分布** | 均匀 / 泊松 / 正态(午间峰值) | 泊松 | **均匀**：每分钟人流恒定；**泊松**：带随机波动；**正态**：钟形曲线分布（12:00为峰值），最符合真实食堂规律 |
| **一楼选择概率** | 10% — 90%（步长10%） | 50% | 学生选择一楼就餐的概率，影响楼层分流。可根据实际调研数据调整 |

#### 2.1.2 仿真控制按钮

| 按钮 | 图标 | 功能 | 使用说明 |
|------|:--:|------|---------|
| **启动仿真** | 🚀 | 开始仿真 | 点击后按照当前参数启动仿真引擎。启动前请确认参数设置正确。仿真运行期间部分参数不可修改 |
| **暂停** | ⏸️ | 暂停仿真 | 暂停正在运行的仿真，所有统计数据保持不变。可点击"继续"恢复 |
| **继续** | ▶️ | 从暂停点恢复 | 从上次暂停的时间点继续运行仿真 |
| **停止** | ⏹️ | 提前结束仿真 | 在仿真自然结束前提前终止。当前统计数据会被保存 |
| **重置** | 🔄 | 清空所有数据 | 清除本次仿真的全部统计数据，包括排队计数、座位占用状态、流量历史等。重置后可重新配置参数 |

**仿真运行状态说明**：

| 状态 | 含义 | 可用操作 |
|------|------|---------|
| 未运行 | 仿真引擎空闲 | 启动、修改参数 |
| 运行中 | 仿真正在进行 | 暂停、停止 |
| 已暂停 | 仿真被暂停 | 继续、停止 |
| 已结束 | 仿真自然结束或手动停止 | 查看结果、重置、查看历史记录 |

#### 2.1.3 窗口管理区（页面右半部分）

显示全部 **16 个窗口**的表格列表，每列说明如下：

| 列名 | 说明 |
|------|------|
| **窗口名称** | 窗口标识（如"面食窗口""家常菜窗口1"等） |
| **平均打饭时间** | 单位为仿真秒，数值越小打饭越快 |
| **吸引力** | 1-10 的整数，值越大越吸引学生排队，影响排队选择权重 |
| **楼层** | 1 楼或 2 楼 |
| **状态** | 开启/关闭。关闭的窗口学生不会排队 |
| **操作** | 编辑、删除按钮 |

**窗口管理操作**：

- **新增窗口**：点击"新增窗口"按钮，填写名称、打饭时间、吸引力、楼层、状态后保存
- **编辑窗口**：点击某行的"编辑"按钮，修改信息后保存
- **删除窗口**：点击某行的"删除"按钮，确认后删除。注意：至少保留一个窗口，否则仿真无法运行

### 2.2 仿真监控页

**访问路径**：`/monitor`

仿真运行时展示实时可视化数据，数据每 **2 秒**自动刷新。本页面包含四个可视化组件：

#### 2.2.1 📊 入场统计组件

| 显示指标 | 含义 |
|---------|------|
| **累计入场人数** | 自仿真开始以来到达食堂的学生总人数 |
| **当前在场人数** | 当前正在食堂内的学生人数（含排队 + 用餐中） |
| **当前占用座位** | 当前被占用的座位数 / 总座位数（514） |

#### 2.2.2 🗺️ 座位热力图

使用 **CSS Grid** 渲染全部 514 个座位的实时占位状态，分为两个楼层：

| 楼层 | 座位数 | 颜色含义 |
|:--:|:----:|---------|
| 1 楼 | 280 座 | 🟢 绿色 = 空闲座位 |
| 2 楼 | 234 座 | 🔴 红色 = 已被占用 |

鼠标悬停可查看具体座位的行列坐标和占用状态。

#### 2.2.3 📈 流量图表

使用 **ECharts** 渲染，包含以下元素：

| 图表元素 | 说明 |
|---------|------|
| **直方图** | 横轴为仿真分钟数，纵轴为该分钟到达的学生人数 |
| **趋势线** | 覆盖在直方图上的平滑曲线，反映人流变化趋势 |
| **到达分布曲线** | 当选择正态分布时，呈现明显的钟形曲线（峰值在12:00附近） |

#### 2.2.4 📋 窗口排队表格

| 列名 | 说明 |
|------|------|
| **窗口名称** | 窗口标识 |
| **排队人数** | 当前在该窗口排队的学生数量，含进度条可视化 |
| **状态** | 窗口是否开启 |
| **楼层** | 1 楼 / 2 楼 |

排队数据每 2 秒刷新一次，方便实时观察各窗口的负载分布。

### 2.3 历史记录页

**访问路径**：`/records`

每次仿真结束（自然结束或手动停止）后，系统自动保存仿真结果到此页面。

#### 2.3.1 记录列表

列表展示所有历史仿真记录，每列说明：

| 列名 | 说明 |
|------|------|
| **编号** | 记录自增 ID |
| **平均等待时间** | 所有窗口所有学生的平均排队等待时间（仿真秒） |
| **座位利用率** | 仿真期间座位的平均占用率（0-100%） |
| **拥堵级别** | 系统自动评估：🟢 畅通 / 🟡 轻度拥堵 / 🟠 中度拥堵 / 🔴 严重拥堵 |
| **仿真时间** | 记录创建的日期和时间 |

#### 2.3.2 查看详情

点击某条记录的"**详情**"按钮，弹出详情窗口，展示：

| 详情内容 | 说明 |
|---------|------|
| **每窗口统计** | 每个窗口的平均排队人数、最高排队人数、平均等待时间、最高等待时间 |
| **拥堵分析** | 系统自动生成的拥堵评估结论 |
| **优化建议** | 基于仿真数据自动生成的分析建议（如"建议午餐时段增加2个窗口""建议提高XX窗口打饭速度"等） |

#### 2.3.3 删除记录

点击某条记录的"**删除**"按钮，确认后从数据库中移除该条仿真记录。

## 三、系统参数配置说明

### 3.1 需要用户手动配置的参数

| 参数 | 文件位置 | 行号 | 说明 |
|------|---------|:--:|------|
| **数据库密码** | `canteensimulation/src/main/resources/application.yml` | 约第12行 | MySQL root 用户的登录密码 |
| **后端端口** | `canteensimulation/src/main/resources/application.yml` | 约第2行 | HTTP 服务端口，默认 8080，如被占用可修改 |
| **前端代理地址** | `canteen-frontend/vite.config.js` | 代理配置部分 | 前端开发服务器代理到后端，默认 `http://localhost:8080` |

### 3.2 数据库密码配置

编辑 `canteensimulation/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/canteen_db?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: ${DB_PASSWORD:你的MySQL密码}   # ← 修改此处为实际密码
```

**推荐使用环境变量**（避免密码写入文件）：

```bash
# Windows PowerShell
$env:DB_PASSWORD="你的密码"

# Windows CMD
set DB_PASSWORD=你的密码

# Linux / macOS
export DB_PASSWORD="你的密码"
```

设置环境变量后，即使 `application.yml` 中未修改默认值，程序也会优先使用环境变量的值。

### 3.3 无需手动配置的项目（程序自动处理）

| 项目 | 自动处理方式 |
|------|------------|
| 数据表结构（t_seat / t_window / t_simulation_record） | 首次启动后端时，`DataInitializer.java` 自动建表 |
| 座位数据（514条） | 首次启动时自动填充，严格按北交大四食堂实际布局 |
| 窗口数据（16条） | 首次启动时自动填充，严格按实际窗口信息 |
| CORS 跨域访问 | `WebMvcConfig.java` 已配置允许前端跨域访问 |
| 前端 API 代理 | `vite.config.js` 已配置将 `/api` 请求代理到后端 |
| MyBatis 数据库映射 | 驼峰命名自动转换、主键自增、SQL 日志已配置 |

## 四、典型使用场景

### 场景一：测试午餐高峰拥堵情况

1. 仿真设置页 → 选择**午餐**时段
2. 设置人员流量为 **120 人/分钟**
3. 速度倍率选 **60x**（快速得出结果）
4. 到达分布选 **正态(午间峰值)**
5. 点击 🚀 **启动仿真**
6. 切换到监控页观察实时数据
7. 仿真结束后查看历史记录详情，获取拥堵评估和优化建议

### 场景二：对比不同窗口配置效果

1. 先按默认配置运行一次仿真，保存记录
2. 在窗口管理区修改窗口吸引力或打饭时间
3. 再次运行相同参数的仿真
4. 在历史记录页对比两次结果的平均等待时间和拥堵级别

### 场景三：早餐/晚餐人流分析

1. 选择对应时段（早餐 7:00-9:00 或 晚餐 17:00-19:00）
2. 调整人员流量（早餐建议 30-80，晚餐建议 40-100）
3. 选择不同的到达分布观察人流模式差异
